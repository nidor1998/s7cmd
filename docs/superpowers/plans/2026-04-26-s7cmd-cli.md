# s7cmd Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `s7cmd`, a single-binary unified S3 CLI bundling s3sync's `sync` plus s3util-rs's 18 subcommands, by vendoring each lib's `src/bin/` code verbatim into s7cmd.

**Architecture:** s7cmd's CLI flattens each lib's existing arg structs into a top-level `Subcommand` enum. Each subcommand's runner, ctrl-c handler, tracing init, indicator UI, and exit-code mapping are vendored verbatim from the upstream lib's `src/bin/` tree (only program name and a handful of `crate::` paths swapped). No global options; no upstream lib modifications.

**Tech Stack:** Rust 2024, Clap 4.6 (derive), Tokio (full), anyhow, tracing/tracing-subscriber, AWS SDK for Rust (transitive via s3sync/s3util-rs), MinIO for local e2e (optional).

**Spec:** `docs/superpowers/specs/2026-04-26-s7cmd-cli-design.md`

## Reference: Vendoring conventions

Apply uniformly to every vendored file unless a task says otherwise.

1. **Source pin.** `s3sync = 1.57.1`, `s3util-rs = 0.2.0`. Local upstream checkouts assumed at `/Users/developer/ghq/github.com/nidor1998/s3sync` and `…/s3util-rs`. If those aren't present, run `cargo install cargo-download && cargo download -x s3sync==1.57.1 -o /tmp/s3sync.tar && tar xf /tmp/s3sync.tar -C /tmp/` (and similarly for s3util-rs); paths below adapt accordingly.

2. **Header on every vendored file** — prepend a comment block:
   ```rust
   // Vendored from <crate>@<version>
   //   <upstream relative path>
   // Adjustments: <comma-separated list, e.g. "stripped #[cfg(test)] mod tests; rewrote crate::cli → super">
   ```

3. **Strip vendored test modules.** Delete the `#[cfg(test)] mod tests { … }` block at the bottom of every vendored file. (s7cmd's own tests live under `tests/`; vendored tests reference upstream-only fixtures and dev-deps we don't carry.)

4. **Path rewrites** (sed search-and-replace inside vendored content):
   - `crate::cli::` → `super::` when the reference is **inside** the same `*_bin/cli/` directory (e.g. `cp.rs` → `super::ExitStatus`).
   - `crate::cli::` → `crate::sync_bin::cli::` (sync_bin) or `crate::util_bin::cli::` (util_bin) when the reference is **outside** the cli/ tree (rare; mostly inside tests we're stripping anyway).
   - `crate::tracing::` → `crate::sync_bin::tracing::` (sync_bin only).
   - `crate::tracing_init::` → `crate::util_bin::tracing_init::` (util_bin only).
   - All `use s3sync::…`, `use s3util_rs::…`, `use aws_sdk_s3::…` etc. stay unchanged — they target the lib deps.

5. **Program-name swap** — only inside s7cmd's `src/main.rs` dispatch arms (NOT inside vendored files):
   - In `clap_complete::generate(...)` calls: `"s3sync"`/`"s3util"` → `"s7cmd"`.
   - The `&mut <X>::command()` argument: replace `s3sync::CLIArgs::command()` and the upstream `Cli::command()` with **s7cmd's** `crate::cli::Cli::command()`.

6. **Verify after every vendored file:** `cargo build` exits 0. If it doesn't, the most likely culprit is a missed `crate::` rewrite or an unstripped test module.

## Reference: Dispatch arm patterns

s3util-rs's `src/bin/s3util/main.rs` uses two patterns by runner return type. Used in Phases 2 and 3.

**Pattern A — runner returns `Result<ExitStatus>`** (cp, mv, create_bucket, head_bucket, head_object, get_object_tagging, get_bucket_tagging, get_bucket_versioning, get_bucket_policy):

```rust
Cmd::Cp(args) => {                                  // ← swap variant + args type
    if let Some(shell) = args.auto_complete_shell() {
        clap_complete::generate(shell, &mut crate::cli::Cli::command(), "s7cmd",
            &mut std::io::stdout());
        return Ok(());
    }
    let config = match s3util_rs::Config::try_from(args) {
        Ok(c) => c,
        Err(msg) => clap::Error::raw(
            clap::error::ErrorKind::ValueValidation, msg).exit(),
    };
    start_tracing_if_necessary(&config);             // helper from main.rs
    trace_config_summary(&config);                   // helper from main.rs
    let exit_code = match crate::util_bin::cli::run_cp(config).await {  // ← swap fn
        Ok(status) => status.code(),
        Err(e) => {
            tracing::error!(error = format!("{e:#}"));
            crate::util_bin::cli::EXIT_CODE_ERROR
        }
    };
    std::process::exit(exit_code);
}
```

**Pattern B — runner returns `Result<()>`** (rm, delete_bucket, put_object_tagging, delete_object_tagging, put_bucket_tagging, delete_bucket_tagging, put_bucket_policy, delete_bucket_policy, put_bucket_versioning):

```rust
Cmd::Rm(args) => {                                  // ← swap variant + args type
    if let Some(shell) = args.auto_complete_shell() {
        clap_complete::generate(shell, &mut crate::cli::Cli::command(), "s7cmd",
            &mut std::io::stdout());
        return Ok(());
    }
    let tracing_config = args.common.build_tracing_config();
    if let Some(tc) = &tracing_config {
        crate::util_bin::tracing_init::init_tracing(tc);
    }
    let client_config = args.common.build_client_config();
    let exit_code = match crate::util_bin::cli::run_rm(args, client_config).await {  // ← swap fn
        Ok(()) => crate::util_bin::cli::EXIT_CODE_SUCCESS,
        Err(e) => {
            tracing::error!(error = format!("{e:#}"));
            crate::util_bin::cli::EXIT_CODE_ERROR
        }
    };
    std::process::exit(exit_code);
}
```

The two helpers `start_tracing_if_necessary(&Config)` and `trace_config_summary(&Config)` come from s3util-rs's `src/bin/s3util/main.rs`; we vendor them into s7cmd's `src/main.rs` once during Phase 2 and reuse for every Pattern A arm.

### Task 1.7: Wire the Sync dispatch arm in main.rs

**Files:**

- [ ] **Step 1: Replace `src/main.rs`**

```rust
// s7cmd entry point. Dispatch arms for each subcommand are direct
// transcriptions of the corresponding match arms in
//   s3sync@1.57.1/src/bin/s3sync/main.rs   (sync arm)
//   s3util-rs@0.2.0/src/bin/s3util/main.rs (all other arms)
// Adjustments: program name "s3sync"/"s3util" → "s7cmd";
//              Cli::command() refers to s7cmd's Cli, not the upstream's.

use anyhow::Result;
use clap::{CommandFactory, Parser};
use clap_complete::generate;

mod cli;
mod sync_bin;

use cli::{Cli, Cmd};

#[tokio::main]
async fn main() -> Result<()> {
    let cli_args = Cli::parse();
    match cli_args.command {
        // ── port of s3sync's main.rs ───────────────────────────
        Cmd::Sync(boxed_args) => {
            let mut config = match s3sync::Config::try_from(*boxed_args) {
                Ok(c) => c,
                Err(msg) => clap::Error::raw(
                    clap::error::ErrorKind::ValueValidation, msg).exit(),
            };
            // s3sync's main: when reporting sync status, force dry_run=true.
            if config.report_sync_status {
                config.dry_run = true;
            }
            if let Some(shell) = config.auto_complete_shell {
                generate(shell, &mut Cli::command(), "s7cmd",
                    &mut std::io::stdout());
                return Ok(());
            }
            if let Some(tc) = &config.tracing_config {
                sync_bin::tracing::init_tracing(tc);
            }
            // sync_bin::cli::run handles ctrl-c, pipeline, indicator, and
            // exits the process with EXIT_CODE_WARNING (3) on warning.
            // Errors propagate up; anyhow → main returns Err → exit 1.
            sync_bin::cli::run(config).await
        }
    }
}
```

# s7cmd

Unified S3 CLI bundling [s3sync](https://github.com/nidor1998/s3sync) and
[s3util-rs](https://github.com/nidor1998/s3util-rs) into a single binary.

## Subcommands

| Subcommand | What it does | Source library |
|---|---|---|
| `sync` | Synchronize a directory tree between local and S3 | s3sync |
| `cp` / `mv` / `rm` | Copy / move / delete objects | s3util-rs |
| `create-bucket` / `delete-bucket` / `head-bucket` | Bucket lifecycle | s3util-rs |
| `head-object` | Object metadata (HEAD) | s3util-rs |
| `get-object-tagging` / `put-object-tagging` / `delete-object-tagging` | Object tagging | s3util-rs |
| `get-bucket-tagging` / `put-bucket-tagging` / `delete-bucket-tagging` | Bucket tagging | s3util-rs |
| `get-bucket-policy` / `put-bucket-policy` / `delete-bucket-policy` | Bucket policy | s3util-rs |
| `get-bucket-versioning` / `put-bucket-versioning` | Bucket versioning | s3util-rs |

Each subcommand accepts the same flags as its standalone equivalent
(`s3sync ...` for `sync`; `s3util <cmd> ...` for the rest). See
`s7cmd <subcommand> --help` for the full flag reference.

## Install

```bash
cargo install s7cmd
```

## Lua scripting

`s7cmd sync` includes s3sync's Lua scripting passthrough by default
(`--filter-callback-lua-script`, `--event-callback-lua-script`,
`--preprocess-callback-lua-script`, etc.). To opt out, build with
`--no-default-features` after re-pinning s3sync with
`default-features = false` in your Cargo.toml.

## License

Apache-2.0. See `LICENSE` and `ATTRIBUTIONS.md` for the upstream sources
that s7cmd vendors.
